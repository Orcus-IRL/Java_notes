Generalization and Specialization

Generalization:

  • The term smartphone came after creating the iPhone, Samsung, vivo. So, the definition is bottom to up.
  • If you have a separate class for iPhone, Samsung and vivo then you can group them together to create a super class.
  • Generalization is achieved using Interfaces.

Specialization:

  • The iPhone X was already there, and iPhone XS was introduced a year after so the definition here is top to bottom.
  • Specialization is achieved using Inheritance.

Abstract classes are used when you need to use both generalization and specialization.

Inheritance:
  • Definition: it’s the process of acquiring the features of an existing class to a new class.
  • Extends keyword is used to inherit from another class.

Example: class cylinder extends circle

  • Class cylinder → subclass, class circle → superclass
Dynamic Method Dispatch
  • Example:

  • Explanation: Car c = new LuxaryCar(); à Dynamic Method Dispatch
  • The methods in super class and the overriding methods in the subclass can be called.
  • But the methods which are unique to sub class cannot be called.
  • When changeGear() is called, the output will display based on the object created (output: automatic gear).

Points to remember for DMD:

  • DMD is useful for achieving run time polymorphism using method overriding.
  • The methods that are called using the reference variable depends on the super class.
  • Which means only super class methods can be accessed by the variable and if a sub class have an overriding method, then it is accessible as well.
  • But the methods which are unique in the sub class cannot be accessed by the reference variable of super class.
Method Overriding

Method Overriding can also be done using Dynamic Method Dispatch.

  • Definition: It is redefining the method of super class in subclass
  • Example: superclass reference name and subclass object name
  • Super su = new Sub() → this line is called as Dynamic Method Dispatch.
  • The method will be called depending upon the object not by the reference, hence su.display() will return “Hello welcome”.
  • while using DMD only super class methods, overriding methods can be called. The unique methods of subclass cannot be called.
  • This is possible but using a sub class for creating a reference and super for creating an object is not possible.
  • Super su = new Sub() is possible, but Sub su = new Super() is not possible .

Do’s and Don’ts when overriding
  • Both the signature should match everything, the datatype of arguments , the return type and all.
  • The access modifier of the subclass can be protected or public, it cannot be private when super class is public.
