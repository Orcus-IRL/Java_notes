static nested class
  • only the inner class can be make static.
  • if you have multiple static method then it can be grouped together to make a static class (inner class) inside outer class.
  • In Java, a static nested class is a nested class that is declared with the static keyword. It is a nested class that belongs to the outer class rather than an instance of the outer class.
  • A static nested class can be used to group a lot of data that is related to the outer class. It is commonly used when there are multiple static methods within the outer class.
  • To declare a static nested class, use the following syntax:
public class OuterClass {
    // Outer class members and methods

    public static class StaticNestedClass {
        // Static nested class members and methods
    }
}
  • The static nested class can be accessed using the outer class name, as shown below:
OuterClass.StaticNestedClass nestedObject = new OuterClass.StaticNestedClass();
  • This allows you to access the members and methods of the static nested class.

example: 

public class OuterClass {
    private static String outerVariable = "Hello from OuterClass";

    public static class StaticNestedClass {
        public void printMessage() {
            System.out.println(outerVariable);
        }
    }
}

public class Main {
    public static void main(String[] args) {
        OuterClass.StaticNestedClass nestedObject = new OuterClass.StaticNestedClass();
        nestedObject.printMessage();
    }
}

In this example, we have an outer class called OuterClass and a static nested class called StaticNestedClass. The StaticNestedClass has a method called printMessage() which prints the value of the outer class's static variable outerVariable.

To create an instance of the static nested class and call its method, we use the syntax OuterClass.StaticNestedClass nestedObject = new OuterClass.StaticNestedClass();. Then, we can call the printMessage() method on the nestedObject instance.

When we run the Main class, it will output "Hello from OuterClass" to the console.

static block
  • it is not commonly used in java.
  • this will be the first thing to execute in the code.
  • the static blocks are executed in the order that you defined.
  • use â†’ if you want to setup/initialize something static block is used.

Here is an example of a static block:

class Example {
    static int num;

    static {
        num = 10;
        System.out.println("Static block is executed.");
    }

    public static void main(String[] args) {
        System.out.println("Value of num: " + num);
    }
}

In this example, we have a class called Example with a static variable num and a static block. The static block is executed before the main method. Inside the static block, we initialize the value of num to 10 and print a message.

When we run the program, the static block will be executed first, and then the main method will be called. The output will be:

Static block is executed.
Value of num: 10

Singleton class
  • a class which cannot create more than one object is called as singleton class .
  • many reference variable can be created but all the reference variable will hold the same object.
class Singleton {
    private static Singleton instance = null;

    private Singleton() {
        // private constructor to prevent instantiation
    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        } 
        return instance;
    }

    public void showMessage() {
        System.out.println("Hello, I am a singleton class!");
    }
}

public class Main {
    public static void main(String[] args) {
        Singleton singleton = Singleton.getInstance();
				//many reference can be created but all will hold the same object.
        singleton.showMessage();
    }
}

In this example, we have a Singleton class with a private constructor to prevent direct instantiation. The class has a static getInstance() method which returns the instance of the class. If the instance is null, a new instance is created. Otherwise, the existing instance is returned.

In the Main class, we call the getInstance() method to get the singleton instance and then call the showMessage() method to display a message.

When we run the program, it will output:

Hello, I am a singleton class!

This example demonstrates the implementation of a singleton class in Java, where only one instance of the class can be created.
