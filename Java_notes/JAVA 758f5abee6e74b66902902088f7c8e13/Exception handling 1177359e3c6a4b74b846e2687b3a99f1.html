  • try and catch block are used to return the error in more user friendly way.
  • if two catch block are used then then the 1st catch block should be a sub-class exception and the 2nd catch block should be a super class of the sub-class exception.
  • nested try and catch can also be used.
  • finally block is used at the end of the try -catch block and it is optional.
  • the finally block will execute no matter what, so mostly it is used for clean up process.
  • if a try block has a return statement and finally block is also there, then before returning anything it will execute the finally block and then it will return.
public class Example {
    public static void main(String[] args) {
        System.out.println(divide(10, 2));
        System.out.println(divide(10, 0));
    }

    public static int divide(int a, int b) {
        try {
            return a / b; // Division by zero will throw an ArithmeticException
        } finally {
            System.out.println("Finally block executed.");
        }
    }
}

Output:

5 --> for the first input
Finally block executed. -->second input
Exception in thread "main" java.lang.ArithmeticException: / by zero
    at Example.divide(Example.java:10)
    at Example.main(Example.java:4)

In this example, the divide() method attempts to perform division a / b. In the first call to divide(10, 2), the division is successful, and the result 5 is returned. In the second call to divide(10, 0), an ArithmeticException is thrown due to division by zero.

Regardless of the exception, the finally block is always executed. In this case, the message "Finally block executed." is printed to the console.

Exception class

The Exception class is the superclass of all exception types in Java. It provides methods and properties to handle and manipulate exceptions.

  • getMessage(): This method returns a string that describes the exception. It can be used to retrieve the error message associated with the exception.
  • toString(): This method returns a string representation of the exception object. It includes the class name, the error message, and the stack trace of the exception.
  • printStackTrace(): This method prints the stack trace of the exception to the standard error stream. It provides detailed information about the sequence of method calls that led to the exception.

Here is an example that demonstrates the usage of these methods:

try {
    int result = 10 / 0; // This will throw an ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Error message: " + e.getMessage());
    System.out.println("String representation: " + e.toString());
    System.out.println("Stack trace:");
    e.printStackTrace();
}

Output:

Error message: / by zero
String representation: java.lang.ArithmeticException: / by zero
Stack trace:
java.lang.ArithmeticException: / by zero
    at MyClass.main(MyClass.java:6)

In this example, an ArithmeticException is thrown when dividing by zero. The getMessage() method retrieves the error message "/ by zero". The toString() method returns the string representation of the exception. Finally, the printStackTrace() method prints the stack trace, showing the line number and class name where the exception occurred.

Here is a simple example of creating your own exception class in Java:

class MyCustomException extends Exception {
    public MyCustomException(String message) {
        super(message);
    }
}

class MyClass {
    public static void main(String[] args) {
        try {
            throw new MyCustomException("This is a custom exception"); //to create an error intentionally throw new keyword is used
        } catch (MyCustomException e) {
            System.out.println("Caught custom exception: " + e.getMessage());
        }
    }
}

In this example, we create a custom exception class called MyCustomException by extending the Exception class. The MyCustomException class has a constructor that takes a message as an argument, which is passed to the superclass constructor using the super() keyword.

In the main() method, we throw an instance of MyCustomException with a custom message. The exception is caught in a catch block specifically for MyCustomException, and the message is printed to the console.

Output:

Caught custom exception: This is a custom exception

By creating your own exception class, you can define and handle custom exceptions that are specific to your program's requirements.

Throw vs Throws

In Java, both throw and throws are used in exception handling, but they have different purposes.

throw Statement

The throw statement is used to explicitly throw an exception in a Java program. It is followed by an instance of an exception class or an object that implements the Throwable interface.

Here is a simple example that demonstrates the usage of the throw statement:

public class MyClass {
    public static void main(String[] args) {
        int age = -10;

        if (age < 0) {
            throw new IllegalArgumentException("Age cannot be negative");
        }
    }
}

In this example, the throw statement is used to throw an IllegalArgumentException (built-in checked exception) if the age is less than zero. This is done to handle the invalid input and terminate the program execution.

throws Clause

The throws clause is used to declare that a method may throw one or more types of exceptions. It is used in the method signature, followed by the names of the exception classes separated by commas.

Here is an example that demonstrates the usage of the throws clause:

public class MyFileReader {
    public void readFile(String filename) throws IOException {
        // Code to read a file
    }
}

In this example, the readFile method may throw an IOException if there is an error while reading the file. By declaring throws IOException in the method signature, the caller of this method is aware that they need to handle or propagate this exception.

To handle the exception thrown by a method with a throws clause, the caller can use a try-catch block or declare the exception using throws in its own method signature.

Overall, the throw statement is used to manually throw an exception, while the throws clause is used to declare that a method may throw an exception and to specify the type of exception that can be thrown.
